## 1. システム概要

### 1.1 プロダクト名
JPGovAI — AIガバナンス統合管理プラットフォーム

### 1.2 目的
企業のAIガバナンス対応を自動化するSaaS。診断→ギャップ分析→文書自動生成→承認→モニタリングまでをワンストップで提供。コンサルに500万円払う代わりに月額3万円で規制遵守を実現。

### 1.3 対象ユーザー
- AI利用企業のコンプライアンス/情報セキュリティ担当者
- DX推進部門のマネージャー
- ISO 42001認証取得を目指す品質管理担当者

### 1.4 対応規制（5フレームワーク）
| 規制 | 定義ファイル |
|------|------------|
| METI AI事業者ガイドライン v1.1 | app/guidelines/meti_v1_1.py |
| ISO/IEC 42001:2023 (AIMS) | app/guidelines/iso42001.py, iso42001_detailed.py |
| NIST AI RMF 1.0 | app/guidelines/nist_ai_rmf.py |
| EU AI Act | app/guidelines/eu_ai_act_detailed.py |
| AI推進法（2025年6月施行） | app/guidelines/ai_promotion_act.py |

### 1.5 技術スタック
| レイヤー | 技術 |
|---------|------|
| UI | Streamlit |
| バックエンド | FastAPI (Python) |
| AI | Anthropic API (Claude) |
| DB | SQLAlchemy + SQLite（本番はPostgreSQL移行予定） |
| PDF生成 | fpdf2 |
| デプロイ | Streamlit Community Cloud |

## 3. データモデル

### 3.1 主要Enum

| Enum | 値 |
|------|-----|
| ComplianceStatus | compliant / partial / non_compliant |
| RiskLevel | high / limited / minimal |
| TaskStatus | todo / in_progress / done / blocked |
| ApprovalStatus | pending / approved / rejected |
| IncidentSeverity | critical / high / medium / low |
| IncidentStatus | open / investigating / resolved / closed |
| UserRole | owner / admin / member / auditor / viewer |
| IndustryType | it / finance / manufacturing / healthcare / retail / other |

### 3.2 主要モデル

| モデル | 用途 | 主要フィールド |
|--------|------|--------------|
| AssessmentResult | 診断結果 | id, organization_id, overall_score, maturity_level, category_scores |
| GapAnalysisResult | ギャップ分析結果 | gaps(28件), compliant_count, partial_count, non_compliant_count |
| RequirementGap | 個別Gapデータ | req_id, title, status, current_score, improvement_actions |
| AutoFixResult | 自動修復結果 | requirement_id, generated_documents, tasks, self_check_questions, status |
| GeneratedDocument | 生成文書 | id, title, content, doc_type, status(draft/approved) |
| AISystem | AIシステム台帳 | id, name, ai_type, vendor, risk_level, status |
| Incident | インシデント | id, type, severity, status, description |
| OrganizationMember | チームメンバー | user_id, organization_id, role |
| ApprovalRequest | 承認リクエスト | id, type, status, requested_by, approved_by |

## 6. セキュリティ設計

### 6.1 認証
- JWT (JSON Web Token) ベースの認証
- ファイル: app/auth.py

### 6.2 監査証跡
- SHA-256ハッシュチェーン + Merkle Tree
- 全操作を改竄防止ログに記録
- チェーン検証API: /api/audit/verify
- ファイル: app/services/audit_trail.py

### 6.3 データ保護
- k-anonymity (k>=5): ベンチマークデータで5社未満のセグメントは非表示
- APIキーはSecrets管理（環境変数、ソースコードに含めない）
